阶段2：自动报名 + 自动排课Phase 2: Auto Enrollment + Auto Scheduling

核心原则：采用 membership 制，必须先完成首月付款才允许进入排课；之后系统按月自动扣费，排课由家长自助完成，系统负责锁时间、生成课表和约束老师确认。Core principle: This runs on a membership model. 阶段4：自动续课系统（核心）Phase 4: Auto Renewal System (Core)

核心目标：membership 的自动续费、续课推荐和排课约束都由系统执行，彻底避免“家长没确认却问排课”的问题。Core goal: Let the system run membership auto-renewal, renewal recommendations, and scheduling constraints so “asking for scheduling before confirmation” no longer happens.

Step 12-15：自动续课触发、推荐与 membership 自动续费Step 12-15: Automated Renewal Trigger, Recommendation, and Membership Auto-Renewal

当 remaining_sessions ≤ 4 时，系统自动发起续课流程；结合当前课程、学生表现和学习路径生成推荐课程、推荐理由和课包。随后按 T-4 节、T-2 节、T-1 节自动提醒。如果家长确认继续，系统续上 membership 并按月自动扣费；只有当续费成功时，才允许继续锁定后续时间。When `remaining_sessions ≤ 4`, the system initiates renewal automatically. It generates the next course, rationale, and package based on current course, performance, and learning path. Then it sends automated reminders at T-4, T-2, and T-1 lessons. Once the parent confirms, the system renews the membership and continues monthly auto-charge; only successful renewal allows future slots to be locked.

If a teacher does not confirm within 24 hours, reminders and reassignment happen automatically.

一句话总结One-Line Summary

这套设计的本质不是“把人全拿掉”，而是让系统承担标准动作、节奏控制和硬规则，把人工集中在真正影响转化和体验的少数节点上。The point of this design is not to remove people entirely. It is to let the system carry standard actions, cadence control, and hard rules, while humans focus on the few moments that truly change conversion and experience.

建议将以上流程接入 CRM、membership 支付、排课与教务系统，统一沉淀：Lead 标签、membership 状态、首月付款状态、月度自动扣费结果、Ready Pool、课消进度、风险标记与家长沟通记录。 Recommend integrating this workflow with CRM, membership billing, scheduling, and academic systems to centralize lead tags, membership status, first-payment status, monthly auto-charge outcomes, Ready Pool gating, lesson consumption, risk flags, and parent communication logs.
